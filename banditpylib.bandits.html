class banditpylib.bandits.Bandit(name: str)[source]¶

Bases: abc.ABC

Class for bandit environment

Before a game is run, one has to call reset() to initialize the environment. During each time step, context() will return the current state of the environment. feed() is used to pass the actions to the environment for execution.

class banditpylib.bandits.OrdinaryMNLBandit(preference_params: numpy.ndarray, revenues: numpy.ndarray, card_limit=inf, reward=None, zero_best_reward=False, name=None)[source]¶

Bases: banditpylib.bandits.utils.Bandit

Class for ordinary MNL bandit

Products are numbered from 1 by default. 0 is reserved for non-purchase. The preference parameters are assumed to be between 0 and 1. And it is assumed that the preference parameter for non-purchase is 1.

Parameters

  • revenue – revenue of products

  • card_limit – cardinality constraint of an assortment

  • reward – reward the learner wants to maximize

  • zero_best_reward – whether to set the reward of the best assortment to 0. This is useful when data is too large to compute the best assortment.
